# Store Screenshot Plan

Use these guidelines when generating screenshots for the Google Play Store listing. This plan ensures we avoid copyright strikes and clearly communicate the app's functionality and privacy focus.

## 1. Setup & Privacy Screen (Home)
- **Visual**: The main OverRead Home Screen.
- **Why**: Shows how easy it is to set target languages and emphasizes the "Privacy & Permissions" card, reinforcing trust.

## 2. Floating Action Button (Active Mode)
- **Visual**: The OverRead floating button resting over a generic, non-copyrighted text background (e.g., a Wikipedia page reading in the public domain or a self-created sample text block).
- **Why**: Demonstrates how non-intrusive the tool is.

## 3. Translation Overlay in Action
- **Visual**: A split or before/after of the self-created sample text, showing the original language and then the translated boxes overlaid neatly on top.
- **Why**: The core value proposition of the app.
- **CRITICAL**: Do NOT use a popular Webtoon, Manga, or protected text for this screenshot. Generate a dummy comic panel or use open public domain content to avoid IP strikes.

## 4. On-Device Model Setup
- **Visual**: The UI showing a model being downloaded with a checkmark for "Ready".
- **Why**: Shows that it runs offline after preparation.

## Important Constraints
- Absolute zero copyrighted third-party material.
- No Personal Identifiable Information (PII) on the screen.
- Highlight "Local Processing" and "No Continuous Capture" in screenshot caption text.
